# Merge

## 3-way-merge
- 특징
  - 새로운 커밋이 생성됩니다.
  - 브랜치의 범위가 확장됩니다.
- 작업 진행
  - main: 1, 2, 3, 4
  - chat: 1, 2, 3, 5, 6
- main브랜치에서 git merge chat 명령어를 실행
- base 커밋 3을 기준으로 chat 브랜치의 5, 6 만큼의 차이를
통째로 계산해서 커밋으로 저장합니다.
- 퀴즈
  - merge가 완료된 후 main 브랜치의 커밋 개수: 7
  - merge가 완료된 후 chat 브랜치의 커밋 개수: 5
  - chat 브랜치를 삭제한 후 main 브랜치의 커밋 개수: 7
- 충돌
  - git status를 사용하면 충돌이 난 파일의 목록을 확인할 수 있습니다.
  - 충돌이 난 파일의 충돌을 "수동으로" 해결하고
  - 스테이징 에어리어에 "수동으로" 추가한 후
  - 새로운 커밋을 "수동으로" 생성한다.
